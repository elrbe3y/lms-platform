## 🔐 طبقات الأمان

### 1️⃣ أمان المصادقة والهوية

#### نظام الجهاز الواحد (Single Device Login)
```
✅ كيف يعمل:
1. عند تسجيل الدخول، يُوّلد "بصمة فريدة" للجهاز
2. تُخزّن الجلسة في قاعدة البيانات مع رقم الجهاز
3. إذا سجّل من جهاز آخر:
   - تُنهى الجلسة القديمة فوراً
   - تُنشأ جلسة جديدة
   - يُقطع الاتصال القديم تلقائياً
```

**التنفيذ الفني:**
- ملف: `lib/session-manager.ts`
- آلية: Device Fingerprinting (IP + User-Agent)
- التخزين: PostgreSQL + Redis (للجلسات النشطة)

#### JWT Tokens مع انتهاء الصلاحية
- **Access Token**: صالح لـ 7 أيام
- **Refresh Token**: صالح لـ 30 يوماً
- **التشفير**: HS256 Algorithm
- **Secret**: مخزّن في `.env` (لا يُرفع على Git)

#### كلمات المرور
- **التشفير**: bcrypt مع 10 جولات
- **الحد الأدنى**: 8 أحرف
- **التوصية**: أرقام + حروف + رموز
- **المنع**: كلمات مرور شائعة (123456، password...)

### 2️⃣ أمان الفيديوهات (الحصن السيبراني)

#### أ) Signed URLs (الروابط المشفرة)
```javascript
// مثال على رابط مشفر:
https://platform.com/api/video/stream?
  v=video_id
  &u=user_id
  &exp=1735689600
  &sig=a7f3c2e8d1b4...
```

**كيف يعمل:**
1. عند طلب الفيديو، يُوّلد رابط فريد للطالب
2. الرابط يحتوي على:
   - معرف الفيديو
   - معرف المستخدم
   - وقت الانتهاء (6 ساعات)
   - توقيع رقمي HMAC-SHA256
3. إذا حاول شخص آخر استخدام الرابط:
   - يفشل التحقق من التوقيع
   - يُسجّل النشاط المشبوه

**الملفات:**
- `lib/video-security.ts` > `generateSignedVideoUrl()`
- `app/api/video/stream/route.ts`

#### ب) العلامة المائية الديناميكية

**المواصفات:**
- **المحتوى**: رقم الواتساب + البريد الإلكتروني
- **الحركة**: عشوائية كل 3-5 ثواني
- **الشفافية**: متغيرة (0.3 - 0.6)
- **المنع**: لا يمكن إزالتها بأدوات بسيطة

**مثال:**
```
┌──────────────────────┐
│  📱 01012345678      │
│  📧 student@ex.com   │
└──────────────────────┘
(تتحرك على الفيديو)
```

**الفائدة:**
- إذا صوّر الطالب الشاشة، ستظهر بياناته
- يمكن تتبع المسرّب فوراً

**الملف:** `components/video/DynamicWatermark.tsx`

#### ج) منع أدوات المطورين

**الإجراءات:**
```javascript
// 1. منع النقر الأيمن
contextmenu = (e) => e.preventDefault()

// 2. منع F12 و Ctrl+Shift+I
if (e.key === 'F12' || (e.ctrlKey && e.shiftKey && e.key === 'I')) {
  e.preventDefault()
}

// 3. منع تحديد النص
user-select: none
```

**الملف:** `components/video/SecureVideoPlayer.tsx`

#### د) HLS Streaming مع التشفير

**للاستضافة على Bunny.net:**
```
1. رفع الفيديو على Bunny Stream Library
2. تفعيل Token Authentication
3. توليد HLS manifest (.m3u8) مشفر
4. كل segment (.ts) يتطلب توكن صالح
```

**الفائدة:**
- لا يمكن تحميل الفيديو مباشرة
- كل قطعة (segment) محمية بتوكن

#### هـ) كشف التسجيل

**آلية:**
```javascript
// إيقاف الفيديو عند تصغير النافذة
document.addEventListener('visibilitychange', () => {
  if (document.hidden) {
    video.pause()
  }
})
```

**الفائدة:** صعوبة التسجيل عبر برامج screen recording

### 3️⃣ أمان قاعدة البيانات

#### Prisma ORM
- **الحماية**: منع SQL Injection تلقائياً
- **Parameterized Queries**: جميع الاستعلامات آمنة
- **مثال:**
```typescript
// ❌ خطر (SQL Injection):
db.query(`SELECT * FROM users WHERE email = '${email}'`)

// ✅ آمن (Prisma):
await prisma.user.findUnique({ where: { email } })
```

#### التشفير
- **كلمات المرور**: bcrypt
- **البيانات الحساسة**: AES-256 (عند الحاجة)
- **Tokens**: HMAC-SHA256

#### النسخ الاحتياطي
- **التكرار**: يومي (يُفضل)
- **التخزين**: منفصل عن الخادم الرئيسي
- **الأمر:**
```bash
pg_dump -U user -d lms_db > backup_$(date +%Y%m%d).sql
```

## 🚨 كشف ومنع الهجمات

### 1. حماية من XSS (Cross-Site Scripting)
```typescript
// React تُنظف المدخلات تلقائياً
// لكن عند استخدام dangerouslySetInnerHTML:
import DOMPurify from 'dompurify'
const clean = DOMPurify.sanitize(userInput)
```

### 2. حماية من CSRF (Cross-Site Request Forgery)
```typescript
// استخدام CSRF Tokens في النماذج
<input type="hidden" name="csrf_token" value={token} />
```

### 3. منع Brute Force
```typescript
// بعد 5 محاولات فاشلة:
- قفل الحساب لمدة 15 دقيقة
- إرسال تنبيه للأدمن
```

### 4. كشف أنماط مشبوهة
```typescript
// مراقبة:
- تسجيل دخول من دول مختلفة خلال دقائق
- محاولات وصول غير مصرح بها
- استخدام كود واحد من أجهزة متعددة
```

## 🔧 إعدادات الإنتاج

### قبل النشر

#### 1. تغيير الأسرار
```env
# ❌ لا تستخدم القيم الافتراضية
JWT_SECRET="change-this-to-random-string"

# ✅ استخدم مولد أسرار عشوائية
JWT_SECRET="a7f3c2e8d1b4e9f6c3a0d5b2..."
```

#### 2. تعطيل Debug Mode
```env
NODE_ENV=production
NEXT_PUBLIC_DEBUG=false
```

#### 3. تفعيل HTTPS
```bash
# استخدم Let's Encrypt للحصول على شهادة SSL مجانية
certbot --nginx -d your-domain.com
```

#### 4. جدار ناري (Firewall)
```bash
# السماح بـ HTTPS و SSH فقط
ufw allow 443/tcp
ufw allow 22/tcp
ufw enable
```

# 🔐 دليل نظام التفعيل المحسّن - الإصدار 2.0

## 📚 نظرة عامة

تم تطوير نظام تفعيل محكم مرتبط بمعرف الجهاز (Device ID) مع حماية شاملة ضد التلاعب.

### ✨ التحسينات الجديدة:

1. **✅ إزالة الوضع التجريبي نهائياً** - التطبيق يتطلب تفعيل إلزامي
2. **🔗 ربط الأكواد بالأجهزة** - كل كود مرتبط بجهاز محدد عبر Device ID
3. **🛡️ حماية من التلاعب بالتاريخ** - كشف تلقائي لأي تلاعب بتاريخ النظام
4. **🚫 منع إعادة استخدام الأكواد** - كل كود يُستخدم مرة واحدة فقط
5. **🔄 فحص دوري تلقائي** - كل 5 دقائق للتحقق من صحة الترخيص

---

## 🎯 خطوات التفعيل

### 📱 **للمستخدم النهائي:**

#### **الخطوة 1: الحصول على Device ID**

1. عند فتح التطبيق، ستظهر صفحة التفعيل تلقائياً
2. ستجد **معرف الجهاز (Device ID)** معروضاً في الصفحة:
   ```
   مثال: DVC-1A2B3C4D5E-123456789ABCD
   ```
3. اضغط على زر **"نسخ معرف الجهاز"** ✅
4. **أرسل Device ID للمطور** عبر WhatsApp/Email/Telegram

⚠️ **مهم جداً:** احتفظ بهذا المعرف، ستحتاجه في كل مرة تريد تفعيلاً جديداً!

#### **الخطوة 2: استلام كود التفعيل**

- سيقوم المطور بإرسال **كود تفعيل فريد** مرتبط بجهازك
- الكود يعمل فقط على جهازك ولا يمكن استخدامه على أجهزة أخرى

#### **الخطوة 3: إدخال كود التفعيل**

1. في صفحة التفعيل، **الصق الكود** في الحقل المخصص
2. اضغط على **"تفعيل الآن"** 🔑
3. إذا كان الكود صحيحاً: ✅ سيتم التفعيل فوراً
4. إذا كان هناك خطأ: ستظهر رسالة توضح السبب

---

### 👨‍💻 **للمطور:**

#### **طريقة 1: مولد HTML (الأسهل)**

1. افتح ملف: `LicenseKeyGenerator/generator.html` في المتصفح
2. **الصق Device ID** الذي استلمته من العميل
3. اختر **نوع الترخيص:**
   - **سنوي:** 365 يوم
   - **مدى الحياة:** لا ينتهي
4. اضغط **"توليد كود التفعيل"**
5. **انسخ الكود** وأرسله للعميل

#### **طريقة 2: Node.js (للمُتقدمين)**

```bash
cd LicenseKeyGenerator
node generator.js <DEVICE_ID>

# مثال:
node generator.js DVC-1A2B3C4D5E-123456789ABCD
```

سيقوم الأمر بـ:
- توليد كود سنوي وكود مدى الحياة
- عرض الأكواد في الـ Terminal
- حفظها في ملف `license.json`

## 🛡️ آليات الحماية المدمجة

### 1️⃣ **الربط بالجهاز**

```javascript
// كل كود يحتوي على Device ID مشفر
licenseKey = Base64(encrypted:signature:deviceId)

// عند التحقق:
if (decoded_deviceId !== current_deviceId) {
  return "هذا الكود غير صالح لهذا الجهاز ❌"
}
```

**النتيجة:**
- ✅ نسخ الكود لجهاز آخر لن يُجدي نفعاً
- ✅ كل جهاز يحتاج كود خاص به

### 2️⃣ **كشف التلاعب بالتاريخ**

```javascript
// النظام يحفظ نقاط زمنية متعددة
systemTimeSnapshot: 1707601234567
lastCheck: 1707601234567

// في كل فحص:
timeDiff = now - lastCheck
systemTimeDiff = now - systemTimeSnapshot

// إذا كان الفرق غير طبيعي:
if (abs(timeDiff - systemTimeDiff) > 5 دقائق) {
  return "تم اكتشاف تلاعب بالتاريخ ❌"
}
```

**ما يتم كشفه:**
- ✅ الرجوع بالتاريخ للخلف
- ✅ القفز بالتاريخ للأمام
- ✅ فجوات زمنية مشبوهة (أكثر من 30 يوم)

**العقوبة:**
- حذف الترخيص فوراً
- إعادة توجيه لصفحة التفعيل
- طلب كود جديد

### 3️⃣ **منع إعادة الاستخدام**

```javascript
// عند التفعيل:
usedLicenses = ["كود1", "كود2", ...]
localStorage.setItem('usedLicenses', usedLicenses)

// عند محاولة إعادة الإدخال:
if (usedLicenses.includes(newCode)) {
  return "هذا الكود مستخدم مسبقاً ❌"
}
```

**النتيجة:**
- ✅ لا يمكن إعادة تفعيل كود محروق
- ✅ حتى بعد حذف البيانات، الكود لن يعمل مرة أخرى

### 4️⃣ **فحص دوري مستمر**

```javascript
// كل 5 دقائق:
setInterval(checkLicense, 5 * 60 * 1000)

// يتم فحص:
- صحة الترخيص
- عدم انتهاء الصلاحية
- صحة التاريخ
- عدم التلاعب بالبيانات
```

## ❌ استكشاف الأخطاء

### **خطأ 1: "هذا الكود غير صالح لهذا الجهاز"**

**السبب:**
- الكود مولد لـ Device ID مختلف
- حاولت استخدام كود شخص آخر

**الحل:**
1. تأكد من Device ID الصحيح (انسخه من صفحة التفعيل)
2. أرسله للمطور مرة أخرى
3. احصل على كود جديد مرتبط بجهازك

---

### **خطأ 2: "هذا الكود مستخدم مسبقاً على هذا الجهاز"**

**السبب:**
- تم تفعيل هذا الكود من قبل على نفس الجهاز
- لا يمكن استخدام الكود أكثر من مرة

**الحل:**
- اتصل بالمطور للحصول على كود تفعيل جديد
- لا يمكن إعادة استخدام الكود القديم أبداً

---

### **خطأ 3: "تم اكتشاف تلاعب بتاريخ النظام"**

**السبب:**
- تم تغيير تاريخ الجهاز يدوياً
- محاولة لتمديد الصلاحية بشكل غير قانوني

**الحل:**
1. أرجع تاريخ الجهاز للتاريخ الصحيح (تلقائي)
2. أعد تشغيل التطبيق
3. إذا استمر الخطأ، احصل على كود تفعيل جديد

⚠️ **تحذير:** النظام يسجل محاولات التلاعب!

---

### **خطأ 4: "انتهت صلاحية الترخيص"**

**السبب:**
- الترخيص السنوي انتهى (365 يوم مضت)

**الحل:**
1. تواصل مع المطور للتجديد
2. احصل على كود تجديد جديد
3. أو احصل على ترخيص مدى الحياة

---

### **خطأ 5: "لم يتم تفعيل التطبيق"**

**السبب:**
- أول مرة تشغل التطبيق
- تم حذف بيانات التفعيل

**الحل:**
- هذا طبيعي! اتبع خطوات التفعيل أعلاه

### **المفاتيح السرية:**

```javascript

```

⚠️ **لا تشارك هذه المفاتيح مع أحد!**
⚠️ **لا تضعها في Git أو أماكن عامة!**

### **آلية التشفير:**

```
1. إنشاء JSON: {id, type, expiryDate, deviceId, ...}
2. ترميز Base64: btoa(JSON)
3. إنشاء توقيع: HMAC(encrypted + deviceId)
4. ترميز Device ID: btoa(deviceId)
5. دمج: ${encrypted}:${signature}:${deviceId}
6. ترميز نهائي: btoa(combined)
```

### **آلية التحقق:**

```
1. فك ترميز: atob(key)
2. فصل: [encrypted, signature, encodedDeviceId]
3. فك Device ID: atob(encodedDeviceId)
4. مقارنة Device ID الحالي بالمحفوظ
5. التحقق من التوقيع
6. فك البيانات وقراءة الصلاحية
```
